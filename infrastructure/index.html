The boring work that makes the interesting work possible. Every research project in this portfolio follows the same engineering standards: type-safe, tested, continuously integrated, deployable.
BUILD SYSTEM
Python Research Libraries

Package Management

All Python projects use pyproject.toml (PEP 621) for package metadata, dependencies, and build configuration. No setup.py, no requirements.txt in production — declarative configuration only. Build backend: setuptools with build-system specification.

Type Safety

mypy --strict on all research code. py.typed marker files for PEP 561 compliance. Type stubs for external dependencies where needed. Runtime type checking via beartype on critical paths. The goal: if it type-checks, the mathematical invariants hold.

Testing

pytest with coverage reporting. Statistical tests use seeded randomness for reproducibility. Coverage target: 90%+ on all libraries (conformal-multimodal: 94%). Property-based testing via hypothesis for coverage guarantees — if the test passes for 1000 random inputs, the guarantee likely holds for all inputs.

CI/CD Pipeline

GitHub Actions workflows on every push and PR. Matrix testing: Python 3.9, 3.10, 3.11, 3.12. Steps: lint (ruff), type-check (mypy --strict), test (pytest --cov), coverage report. Branch protection: PRs require passing CI before merge.

FRONTEND
Visualization Infrastructure

WebGL Pipeline (SHOGGOTH)

Raw WebGL 2.0 — no Three.js, no abstractions. Custom GLSL vertex and fragment shaders compiled at runtime. Particle state managed in typed arrays for direct GPU upload. RequestAnimationFrame loop with delta-time physics. Zero dependencies means zero supply chain risk and complete control over the render pipeline.

Static Site Deployment

GitHub Pages for all live demos (CHIMERA, SHOGGOTH). Custom domain (colonel1223.net) via CNAME. No build step — static HTML, CSS, JS served directly. This is intentional: research demos should be inspectable. View source is documentation.

Design System

IBM Plex Mono + Cormorant Garamond. CSS custom properties for consistent theming across all pages. Grain overlay via inline SVG filter (no external assets). Scroll-reveal via IntersectionObserver. Responsive: single-column below 640px. Every page shares the same design language — the portfolio is itself a demonstration of engineering consistency.
